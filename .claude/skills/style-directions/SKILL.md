# Style Directions — an inspiration library, not a template picker

This is a browsable library of 66 named aesthetic directions, each with a one-line pitch, a fuller design brief (`presets/<name>/SKILL.md`), and machine-readable starter tokens (`presets/<name>/DESIGN.md`: colors, type, spacing, radius). Most also have a visual preview at `previews/<name>-marketing.png`.

**How to use this — the point is inspiration, not application:**

1. Pick the 1-3 closest-sounding directions for the brief (e.g. a request for "cyberpunk" maps closest to `matrix`, `neon`, or `cosmic` — none of them *is* cyberpunk, they're the nearest starting points).
2. Read that preset's `SKILL.md` and `DESIGN.md` for its concrete moves (palette logic, type pairing, the shapes/textures it reaches for).
3. **Then diverge.** Cross-pollinate with the actual brief — the client's real content, voice, and subject matter (per the `frontend-design` skill's "ground it in the subject" rule) — and with `taste-skill`'s brand judgment. The preset is a seed, not the deliverable. Shipping a preset's tokens unmodified is exactly the "templated" look the IDP's anti-sameness principle exists to prevent (see `docs/modules/anti-ai-look.md`).
4. If nothing fits, that's a signal to invent rather than force-fit one of the 66.

## Relationship to other skills

- **`frontend-design`** owns the actual design *process* (ground it in the subject, brainstorm/plan/critique/build) — this skill just supplies a faster starting point for the brainstorm step when a vibe is already named.
- **`taste-skill`** is the brand-judgment / anti-laziness pass that should still run after a direction is chosen — a preset picked here is not exempt from that review.
- **`power-design`**'s `brands/` library is different: those are 70+ *real brand systems* (extracted DNA), not abstract style categories. Use `power-design` when there's a specific brand to match; use this skill when exploring an aesthetic *category*.
